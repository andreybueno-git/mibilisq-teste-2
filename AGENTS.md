# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Mibilisq design decisions

- Brand identity is orange, yellow, and white. Green is not part of the default site identity.
- The green/yellow Instagram logo is a temporary World Cup campaign logo. Do not use it as the permanent brand logo.
- Until the permanent logo is supplied, use an editable orange typographic wordmark reading “mibilisq”.
- Visual direction is Vintage Artesanal: warm paper texture, editorial serif display type, food photography, label-inspired details, and restrained print grain.
- Motion is essential: layered hero entrance, collage-style product reveals, soft parallax, animated category labels, and tactile cart feedback.
- Motion must remain lightweight, simplify on mobile, and respect `prefers-reduced-motion`.
- The first vertical slice contains the public storefront, catalog filters, cart, and WhatsApp handoff.
